Voltage Engine · Audio System

A full-stack game audio system, from one-shot to reverb tail.

Everything you need to score a 2D game with heavy atmosphere — a bussed mixer, 2D spatial sound, designer-placed zones for ambience, crowds, music and mood, real voice management, and an optional software-mixing backend with DSP for occlusion and reverb. Built on MonoGame, NativeAOT-safe, pure-managed decoders.

01 — SOURCES

Sources & playback

One component covers the common cases: fire-and-forget SFX, looping ambience, and 2D positional sound — all live-tunable from the inspector while the game runs.

One-shots & loops

Play a clip once or loop it as a tracked, controllable voice. Overlapping SFX go fire-and-forget.

2D positional

Attenuate and pan by distance to the listener, with an inner full-volume radius and an outer silence radius.

Pitch · pan · volume

Standard per-voice controls, synced live every frame — inspector edits are heard instantly in Play mode.

Volume · Pitch · Pan

Fades & crossfade

Priority & caps

02 — ATMOSPHERE

Zones & atmosphere

Drop a trigger volume, dial the levels, done. These designer-facing components author a soundscape along a linear story without a line of code — the backbone for crowded street scenes.

Ambience beds

Looping environmental beds that crossfade in on enter and out on exit. Overlapping zones blend, so a street tone layers under a fountain.

AmbienceZoneComponent

Crowd walla

Scatters randomized murmurs, coughs and footsteps around a point with a density knob — a living crowd from one emitter, not fifty sources.

CrowdEmitterComponent · Density

Location music

Start a track when the player enters a region; it crossfades from whatever was playing.

MusicZoneComponent

Mixer moods

Shift the whole mix to a "mood" on enter — pull music up, drop the crowd — as a smooth per-bus crossfade.

MixerSnapshotZoneComponent

Dialogue ducking

While anything plays on the Voice bus, music & ambience automatically dip so speech stays clear, then recover.

DialogueDuckingComponent

Room reverb

Make a room, tunnel or cathedral reverberant while inside a zone. Software backend only; a no-op elsewhere.

ReverbZoneComponent

04 — SPATIAL & VOICES

Spatial & voice management

2D spatialization keeps crowded scenes legible, and a real voice manager keeps them from blowing past the platform's ~32-source ceiling.

Distance & pan

Hand-computed linear attenuation between Min/Max distance plus a stereo pan from the emitter's offset — predictable, full control.

AudioListenerComponent

Priority stealing

When the pool is full, a newcomer steals the weakest voice only if it outranks it — otherwise it waits or drops.

MaxVoices

Distance culling

Sounds beyond audible range never take a real voice; inaudible one-shots are dropped before they cost anything.

IsAudible

Virtual voices

A far-away loop goes "virtual" — tracked but silent, costing no hardware voice — and revives seamlessly when back in range.

IsVirtual · revive

Range gizmos

Every 3D source draws its reach in the editor: green = full volume, orange = silence, cyan = crowd scatter radius.

DebugRender

Live tuning

Edit any audio component in Play mode — even while the editor is paused to inspect — and hear the change immediately.

IUpdatableInPauseMode
05 — BACKEND & DSP

Backends & DSP

One swappable seam under everything. The default plays through MonoGame; an opt-in software mixer owns the PCM loop to unlock effects the per-sound path can't do — and falls back gracefully where it isn't supported.

Two backends

MonoGame per-voice playback (default, everywhere) or a software mixer that sums every voice into one dynamic stream.

IAudioBackend

Clip abstraction

Components deal only in AudioClip, so the clip representation — SoundEffect or decoded PCM — changes with the backend, not the code.

AudioClip · LoadClip

Capability fallback

The software backend probes the platform and falls back to MonoGame if unsupported. DSP simply no-ops there — nothing breaks.

IsSupported

Occlusion low-pass

A per-voice low-pass cutoff muffles sounds behind walls or far away — the DSP hook the per-instance path never had.

LowPassCutoffHz

Reverb send

A shared Freeverb (8 comb + 4 all-pass); each voice feeds it by a send amount, driven by reverb zones.

ReverbSend · SetReverb

Managed decoders

wav (RIFF), ogg (NVorbis) and mp3 (NLayer) decoded in pure managed code — no native binaries, AOT- and trim-safe.

AudioDecoders
TUNING

Cost controls

Every place that can bound CPU or the voice pool is an explicit, tunable knob — no silent caps. Dial these per title, especially for mobile.

The knobs that bound cost

MaxVoices — real voice ceiling MaxInstances — per-clip cap MaxConcurrent — crowd bits Density — crowd spawn rate SetReverbEnabled — DSP toggle SIMD — vectorized output

voltage.audio — MonoGame · .NET 8 · NativeAOT-safe. Default backend is MonoGame; the software mixing backend (DSP: occlusion + reverb) is opt-in via AudioManager.PreferSoftwareBackend and recommended on desktop. Streaming music and mobile profiling are the remaining follow-ups.
